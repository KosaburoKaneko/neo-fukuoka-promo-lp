# AGENTS.md

## 役割

あなたは、ネオフクオカ公式オンラインストアの販促LPを制作・改善するコーディングエージェントです。

このファイルに記載された要件を最優先で読み、必要なディレクトリ、ファイル、画像取得スクリプト、LP本体、スタイル、トラッキング処理、商品データ、キャンペーン設定、レポートを作成・更新してください。

このプロジェクトは、父の日専用ではありません。父の日、母の日、敬老の日、お中元、お歳暮、年末年始、夏のそうめん訴求、冬の鍋・ごはん訴求、新商品発売、LINE配信、Instagram投稿、Meta広告など、季節・イベント・キャンペーンに応じた販促LPを制作するための汎用プロジェクトです。

## 最重要方針

LPの目的は、ネオフクオカの商品を、季節・イベント・配信内容に合わせて訴求し、公式オンラインストアでの購入につなげることです。

商品説明だけでなく、食卓での使い方や利用シーンを伝えてください。

ネオフクオカの基本トーンは、以下です。

```txt
いつものごはんを、ちょっと楽しく。
```

この方向性を基準に、キャンペーンごとの文脈に合わせてコピー、構成、CTA、商品導線を設計してください。

## 商品URL・商品画像取得ルール

商品URLと商品画像は、公式オンラインストア上から取得してください。

取得対象商品は、キャンペーンごとに変わります。固定の商品リストだけを前提にしないでください。

取得すべき情報は以下です。

* 商品名
* 商品URL
* 商品画像URL
* ローカル保存画像パス
* 価格
* 内容量
* 商品説明
* 対象キャンペーン
* CTA用識別子

取得できた商品URL、商品名、画像URL、ローカル保存画像パスは、原則として以下に保存してください。

```txt
neo-fukuoka-promo-lp/data/products.json
```

商品URLや画像の取得に失敗した場合は、仮リンクや仮画像で済ませず、以下に不足内容を記載してください。

```txt
reports/missing-assets.md
```

## 商品画像の扱い

公式オンラインストア上から取得できる商品画像を使用してください。

キャンペーンに応じて、以下のような画像を使用します。

* 商品単体画像
* セット商品画像
* 商品が並んだ画像
* 食べ方や利用シーンの画像
* ギフト訴求の場合はギフトボックス画像
* 季節訴求の場合は季節感のある使用シーン画像

不足しやすい画像は、`reports/missing-assets.md` に以下のように記載してください。

```md
# 不足画像リスト

- 対象商品
- 必要なセクション
- 取得を試みたURLまたは対象商品名
- 代替案が必要な場合のメモ
```

## LPの目的

LPの目的は、ネオフクオカの商品をキャンペーン文脈に合わせて訴求し、公式オンラインストアでの購入につなげることです。

具体的には以下を重視してください。

* 初見の人にも何の商品か分かること
* 食べ方や使うシーンが伝わること
* キャンペーンの文脈と商品訴求が自然につながること
* 公式オンラインストアの商品ページへ迷わず遷移できること
* スマートフォンで読みやすく、購入導線が分かりやすいこと
* LP経由の流入・購入をUTMで分析できること

## ターゲット

ターゲットは、キャンペーンごとに設定してください。

想定例：

* 公式LINEから訪れる既存顧客
* Instagram投稿・ストーリーズから訪れるユーザー
* Meta広告から訪れる新規ユーザー
* 催事で商品を知った後にECで購入を検討するユーザー
* 季節の食べ方を探しているユーザー
* ギフト商品を探しているユーザー
* 以前購入した商品の再購入を検討しているユーザー

ターゲットに合わせて、コピーの温度感、CTA、FAQ、商品説明の粒度を調整してください。

## LP全体の訴求方針

基本方針は以下です。

* 商品そのものだけでなく、食卓での使い方を伝える
* 「いつものごはんを、ちょっと楽しく」というブランド感を保つ
* 季節・イベントの文脈に自然に接続する
* 初見でも分かるように、商品説明を省略しすぎない
* 既存顧客には、使い方や組み合わせの提案で再購入理由を作る
* 過度に煽らない
* 健康効果や効能をうたわない
* 断定的すぎる表現を避ける
* 商品ページの実際の情報と矛盾しない

## LP構成の基本型

キャンペーンごとに変更して構いませんが、基本構成は以下を推奨します。

1. ファーストビュー
2. キャンペーン訴求・導入
3. 商品をおすすめする理由
4. 商品別おすすめシーン
5. 商品購入エリア
6. おすすめの食べ方・使い方
7. FAQ
8. 最下部CTA

ギフト訴求の場合は、以下を追加しても構いません。

* ギフトボックス訴求
* 贈り物向けFAQ
* 配送・包装・同梱物に関する注意事項

食べ方訴求の場合は、以下を強化してください。

* 料理別の使い方
* 使用シーン画像
* 商品ごとのおすすめ料理
* 味の違い
* セット購入理由

必要な要素は以下です。

* 購入ボタン

## FAQ

FAQはキャンペーンごとに作成してください。

汎用FAQの例：

### Q. どの商品を選べばいいですか？

A. キャンペーン内容や食べ方に合わせておすすめ商品を掲載しています。迷った場合は、商品購入エリアの説明やおすすめ対象を参考にしてください。

### Q. 公式オンラインストアで購入できますか？

A. はい。各商品のボタンから、ネオフクオカ公式オンラインストアの商品ページへ移動できます。

### Q. 商品画像や価格は最新ですか？

A. 商品URL、商品画像、価格は公式オンラインストアの情報をもとに掲載してください。実装時点で確認できない情報は断定しないでください。

ギフト訴求の場合は、配送先指定、包装、のし、メッセージカード、同梱物などのFAQを追加して構いません。ただし、未確認の対応は断定しないでください。

## デザイン要件

### トーン

* 食品としての清潔感
* ネオフクオカらしい少し新しい雰囲気
* 高級すぎず、カジュアルすぎない
* 季節感は出すが、チラシ感を強くしすぎない
* 商品の使い方が直感的に伝わる

### 色味

キャンペーンごとに調整して構いません。

基本候補：

* 白
* 生成り
* 薄いベージュ
* 深い緑
* 赤
* 黒
* 落ち着いた黄色
* ネイビー
* 金系の控えめなアクセント

避けたい色味：

* 子どもっぽいポップすぎる色
* スーパーのチラシ感が強い赤
* ギラギラした金色
* 商品イメージと合わない季節色
* 辛い商品だけに見える赤一色の構成

### レイアウト

* スマートフォン閲覧を最優先
* ファーストビューにCTAを設置
* 商品カードはタップしやすいサイズにする
* 商品写真と食べ方写真を大きめに表示
* CTAボタンは各主要セクション後に設置
* 商品購入エリアまでのスクロール距離を長くしすぎない
* PC表示でも崩れないレスポンシブ対応にする

BASE側で流入元を特定できるように、CTAには必ずUTMパラメータを付与してください。

* 商品を見る
* セット商品を見る
* 詳しく見る
* 購入ページへ進む
* 今回のおすすめを見る

ギフト訴求向けCTA：

* ギフトセットを見る
* 贈り物に選ぶ
* ギフトボックス付きセットを見る

* この商品を見てみる

## UTM設計

LPへの流入元ごとに、CTA遷移時のUTMをBASE商品ページへ引き継いでください。

### デフォルト値

LPにUTMが付与されていない場合は、以下のデフォルト値を使用してください。

```txt
utm_source=lp
utm_medium=owned
utm_campaign=seasonal_promo
```

`utm_campaign` は固定値としてコードに埋め込まず、キャンペーン設定から変更できるようにしてください。

### 流入元別の例

Meta広告用：

```txt
utm_source=meta
utm_medium=paid_social
utm_campaign={campaign_name}
utm_content=lp_cta
```

Instagram投稿用：

```txt
utm_source=instagram
utm_medium=social
utm_campaign={campaign_name}
utm_content=lp_cta
```

Instagramストーリーズ用：

```txt
utm_source=instagram
utm_medium=stories
utm_campaign={campaign_name}
utm_content=lp_cta
```

LINE配信用：

```txt
utm_source=line
utm_medium=message
utm_campaign={campaign_name}
utm_content=lp_cta
```

### CTA位置別utm_content

CTA位置別に識別できるよう、可能であれば `utm_content` を以下のように出し分けてください。

* `fv_cta`
* `reason_cta`
* `product_card_01`
* `product_card_02`
* `product_card_03`
* `bottom_cta`

### UTM引き継ぎ仕様

LPアクセス時のURLパラメータをJavaScriptで読み取り、CTA遷移時にBASEの商品URLへ引き継いでください。

LPにUTMが付与されている場合は、その値を優先してください。

LPにUTMが付与されていない場合は、キャンペーン設定またはデフォルト値を使用してください。
