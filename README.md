# ネオフクオカ 公式オンラインストア販促LP制作プロジェクト

## 概要

このプロジェクトは、ネオフクオカ公式オンラインストアへの購入導線を強化するための、販促用LPを制作・管理するプロジェクトです。

父の日、母の日、敬老の日、お中元、お歳暮、年末年始、夏のそうめん訴求、冬の鍋・ごはん訴求、新商品発売、LINE配信、Instagram投稿、Meta広告など、季節・イベント・キャンペーンに応じたLPを制作するために使用します。

LPはBASEとは別ドメインで公開し、CTAからネオフクオカ公式オンラインストアの各商品ページへ遷移させます。

## 目的

季節やキャンペーンごとの文脈に合わせて、ネオフクオカの商品を「いつものごはんを、ちょっと楽しくする商品」として訴求し、公式オンラインストアでの購入につなげることを目的とします。

単に商品を並べるだけではなく、以下を重視します。

* 初見の人にも、何の商品か分かること
* 商品の使い方や食卓でのシーンが伝わること
* 季節・イベント・配信内容と商品訴求が自然につながること
* 公式オンラインストアの商品ページへ迷わず遷移できること
* LP経由の流入・購入をUTMで把握できること

## このプロジェクトで作るもの

主に以下を制作・管理します。

* 販促LP本体
* スマートフォン最優先のレスポンシブCSS
* CTAクリック時にUTMを引き継ぐJavaScript
* 商品データ管理用ファイル
* キャンペーン設定ファイル
* 公式オンラインストアから商品URLと商品画像を取得するスクリプト
* 不足画像を記録するレポートファイル
* 必要なディレクトリ・アセット管理構成

## 想定するLP例

このプロジェクトでは、以下のようなLPを想定します。

* 父の日ギフトLP
* 母の日ギフトLP
* 敬老の日ギフトLP
* お中元・お歳暮LP
* 年末年始の手土産・ごはん訴求LP
* 夏のそうめん向けLP
* 冬の鍋・ごはん向けLP
* 新商品発売LP
* LINE配信用LP
* Instagram投稿・ストーリーズ用LP
* Meta広告用LP
* 催事後のEC誘導LP
* 既存顧客向けの再購入促進LP

父の日LPは、このプロジェクトで制作するキャンペーンLPの一例です。プロジェクト全体は父の日専用ではありません。

## 公式オンラインストア

商品URLと商品画像は、以下の公式オンラインストアトップページを起点に取得します。

```txt
https://store.neo-fukuoka.jp/
```

## GitHub Pages 公開方針

GitHub Pages では、キャンペーンごとにサブディレクトリを切って公開します。

```txt
お中元LPの公開URL:
https://kosaburokaneko.github.io/neo-fukuoka-promo-lp/ochugen_2026/
```

公開成果物では、LP本体 `neo-fukuoka-promo-lp/` の中身を `dist/ochugen_2026/` にコピーします。

今後のキャンペーン例:

```txt
ochugen_2026/
fathersday_2026/
```

## 推奨ディレクトリ構成

基本構成は以下です。

```txt
.
├── README.md

├── package.json
├── neo-fukuoka-promo-lp
│   ├── index.html
│   ├── styles
│   │   └── main.css
│   ├── scripts
│   │   ├── main.js
│   │   └── tracking.js
│   ├── data
│   │   ├── products.json
│   │   └── campaign.json
│   └── assets
│       ├── images
│       └── icons
├── scripts
│   └── fetch-store-assets.js
└── reports
    └── missing-assets.md
```

実装環境に合わせて、Vite、Next.js、Astro、静的HTMLなどの構成に変更しても構いません。

ただし、以下は守ってください。

* 静的LPとして公開しやすいこと
* テキストや画像を後から修正しやすいこと
* キャンペーンごとのコピー・商品構成・UTMを変更しやすいこと
* 父の日など特定キャンペーン専用のディレクトリ名を標準構成にしないこと

## 商品データ管理

商品データは、原則として以下で管理します。

```txt
neo-fukuoka-promo-lp/data/products.json
```

商品データには、少なくとも以下を含めます。

* 商品名
* 商品URL
* 画像URL
* ローカル保存画像パス
* 価格
* 内容量
* 簡単な説明文
* おすすめ対象
* CTA用の識別子

商品URLや画像は、公式オンラインストアから取得してください。

取得できない場合は、仮リンクや仮画像で済ませず、`reports/missing-assets.md` に不足内容を記録します。

## キャンペーン設定

キャンペーンごとのコピー、SEO、UTM、CTA文言は、可能であれば以下のファイルで管理します。

```txt
neo-fukuoka-promo-lp/data/campaign.json
```

想定する基本構造は以下です。

```json
{
  "campaign_name": "seasonal_promo",
  "campaign_label": "季節のおすすめ",
  "utm_campaign": "seasonal_promo",
  "page_title": "季節のおすすめ｜ネオフクオカ公式オンラインストア",
  "meta_description": "ネオフクオカ公式オンラインストアの季節限定・販促用LPです。",
  "h1": "いつものごはんを、ちょっと楽しく。",
  "main_copy": "",
  "sub_copy": "",
  "cta_label": "公式オンラインストアで見る"
}
```

キャンペーンごとに変わる主な項目は以下です。

* キャンペーン名
* 対象時期
* 対象商品
* メインコピー
* サブコピー
* CTA文言
* SEOタイトル
* メタディスクリプション
* UTM campaign
* ファーストビュー画像
* 商品の見せ方
* FAQ

## UTM設計

LPはBASEとは別ドメインで制作するため、すべてのCTAからBASE商品ページへ遷移させる際にUTMパラメータを付与します。

デフォルト値は以下を基準にします。

```txt
utm_source=lp
utm_medium=owned
utm_campaign=seasonal_promo
```

ただし、LPアクセス時に以下のUTMが付与されている場合は、その値をCTA遷移時に引き継ぎます。

* `utm_source`
* `utm_medium`
* `utm_campaign`
* `utm_content`
* `utm_term`

`utm_content` はCTA位置ごとに出し分けます。

例：

* `fv_cta`
* `reason_cta`
* `product_card_01`
* `product_card_02`
* `product_card_03`
* `bottom_cta`

父の日など特定キャンペーンのUTMは、固定値としてコードに埋め込まず、キャンペーン設定から変更できるようにします。

## CTA設計

CTAは、BASEの商品ページへ遷移させるために設置します。

汎用的なCTA文言の候補は以下です。

* 公式オンラインストアで見る
* 商品を見る
* セット商品を見る
* 詳しく見る
* 購入ページへ進む
* 今回のおすすめを見る

ギフト訴求の場合は、以下のような文言を使用しても構いません。

* ギフトセットを見る
* 贈り物に選ぶ
* ギフトボックス付きセットを見る

CTAは以下のような位置に設置します。

* ファーストビュー
* 商品訴求セクション後
* 商品カード
* FAQ前後
* LP最下部

## LP構成の基本型

キャンペーンごとに変更して構いませんが、基本構成は以下を推奨します。

1. ファーストビュー
2. キャンペーン訴求・導入
3. 商品をおすすめする理由
4. 商品別おすすめシーン
5. 商品購入エリア
6. おすすめの食べ方・使い方
7. FAQ
8. 最下部CTA

ギフト訴求が必要な場合は、ギフトボックス訴求や贈り物向けFAQを追加して構いません。

夏のそうめん訴求など、食べ方提案が中心の場合は、食べ方・使用シーンを上部に寄せても構いません。

## 商品画像の扱い

商品画像は公式オンラインストア上から取得できるものを使用します。

必要に応じて、以下の画像を使用します。

* 商品単体画像
* セット商品画像
* 使用シーン画像
* 食べ方画像
* 季節・イベントに合うビジュアル
* ギフト訴求の場合はギフトボックス画像

不足しやすい画像は、`reports/missing-assets.md` に記録します。

## 不足画像レポート

商品URLや画像の取得に失敗した場合は、以下に不足内容を記載します。

```txt
reports/missing-assets.md
```

不足画像レポートには、以下を記載します。

* 不足している画像の種類
* どの商品・どのセクションで必要か
* 取得を試みたURLまたは対象商品名
* 代替案が必要な場合のメモ

仮画像や仮リンクで済ませないでください。

## コピーライティング方針

ネオフクオカのブランドトーンに合わせ、以下を守ります。

* 「いつものごはんを、ちょっと楽しく」という方向性を基本にする
* 商品説明だけでなく、使う場面を伝える
* 初見の人にも何の商品か分かるようにする
* 既存顧客にも違和感がない表現にする
* 季節・イベントに合わせた訴求を行う
* 過度に感動的・煽りすぎる表現は避ける
* 健康効果・効能表現は使わない
* 「絶対喜ばれる」「万人におすすめ」など断定的すぎる表現は避ける
* 「日本一」「完全無添加」など事実確認が必要な表現は使わない
* 薬機法・景表法上リスクのある表現は使わない

## デザイン方針

* スマートフォン閲覧を最優先する
* ファーストビューにCTAを設置する
* 商品カードはタップしやすいサイズにする
* 商品写真と食べ方写真を大きめに表示する
* CTAボタンは主要セクション後に設置する
* 商品購入エリアまでのスクロール距離を長くしすぎない
* キャンペーンごとの雰囲気は出す
* ただし、ネオフクオカのブランド感を崩さない
* 季節感を出しすぎて、別ブランドやチラシ風に見えないようにする

## SEO方針

キャンペーンごとに以下を設定します。

* ページタイトル
* メタディスクリプション
* h1
* OGPタイトル
* OGPディスクリプション
* OGP画像

父の日、母の日、お中元など、検索需要があるテーマでは、検索キーワードを自然に含めます。

ただし、検索キーワードを詰め込みすぎず、実際のLP内容と一致させてください。

## 計測要件

可能であれば、CTAクリックイベントを計測できる設計にします。

イベント名の例：

* `click_fv_cta`
* `click_reason_cta`
* `click_product_card_01`
* `click_product_card_02`
* `click_product_card_03`
* `click_bottom_cta`

GA4やMetaピクセル等を導入する場合は、既存の計測設計と矛盾しないようにします。

## 実装完了時に確認すること

* スマートフォン表示でファーストビューが自然に見えること
* CTAが複数箇所に設置されていること
* CTAリンクにUTMが付与されていること
* LP流入時のUTMがCTA遷移時に引き継がれること
* 商品カードからBASEの商品ページへ遷移できること
* 商品画像が取得・配置されていること
* 不足画像がある場合は `reports/missing-assets.md` に記載されていること
* キャンペーンごとのコピー・SEO・UTMを変更しやすいこと
* 父の日など特定キャンペーン専用の構成になっていないこと
* 健康効果、効能、断定的な景表法リスク表現を使っていないこと
